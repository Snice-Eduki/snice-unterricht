# HTML im Unterricht: Materialien, Ideen und Arbeitsblätter für die Klassen 4–7

Lehrkräfte, die das Thema **HTML** im Deutsch- oder Informatikunterricht behandeln möchten, stehen oft vor der Herausforderung, altersgerechte und praxisnahe Materialien zu finden. Besonders in den Klassen 4–7 geht es darum, Grundlagen verständlich zu vermitteln – ohne technische Überforderung. Ein **HTML Arbeitsblatt** mit Lückentext, Wortspeicher und Hörverstehen kann hier ein idealer Einstieg sein, um Schüler:innen spielerisch an die Struktur von Webseiten heranzuführen.

In diesem Artikel erfahren Sie:
- **Hintergrundwissen** zu HTML und seiner Relevanz im Unterricht,
- **konkrete Unterrichtsideen** für Einstieg, Erarbeitung und Sicherung,
- **praktische Methoden** zur Differenzierung und Sprachförderung,
- **Antworten auf häufige Fragen** von Lehrkräften zum Thema.

## Hintergrundwissen: Was ist HTML und warum gehört es in den Unterricht?

### HTML als Grundlage des World Wide Web
HTML (*Hypertext Markup Language*) ist die **Standardsprache für die Erstellung von Webseiten**. Sie wurde Anfang der 1990er-Jahre entwickelt und bildet bis heute das Rückgrat des Internets. Fast jede **Webseite**, die Schüler:innen täglich nutzen – von Suchmaschinen über Lernplattformen bis zu sozialen Medien –, basiert auf HTML.

Im Gegensatz zu Programmiersprachen wie Python oder JavaScript ist HTML **keine Programmiersprache**, sondern eine **Auszeichnungssprache** (*Markup Language*). Das bedeutet:
- Sie beschreibt die **Struktur und Darstellung** von Inhalten (z. B. Überschriften, Absätze, Bilder).
- Sie verwendet **Tags** (z. B. `<h1>`, `<p>`, `<a>`), um Elemente zu kennzeichnen.
- Sie wird vom **Browser** interpretiert und in eine sichtbare Webseite umgewandelt.

Für den Unterricht ist HTML besonders geeignet, weil:
✔ **Einfacher Einstieg**: Die Syntax ist überschaubar und lässt sich mit Grundkenntnissen schnell anwenden.
✔ **Sichtbare Ergebnisse**: Schüler:innen sehen direkt, wie ihr Code eine Webseite verändert.
✔ **Fächerübergreifender Bezug**: HTML verbindet Deutsch (Texte strukturieren), Informatik (digitale Grundbildung) und Medienkompetenz.

### Wichtige HTML-Konzepte für den Unterricht
Um das Thema altersgerecht aufzubereiten, sollten folgende Aspekte vermittelt werden:

#### 1. **Hypertexte und Links**
HTML steht für *Hypertext Markup Language*. Der Begriff **Hypertext** bezeichnet Texte, die mit anderen Texten verknüpft sind – die Grundlage für das **World Wide Web**. Ein zentrales Element sind **Links** (`<a>`-Tag), die es ermöglichen, zwischen Webseiten zu navigieren.

**Beispiel für den Unterricht**:
- Schüler:innen erstellen eine einfache "Link-Sammlung" zu ihren Lieblingswebseiten.
- Sie lernen, wie Links funktionieren (z. B. `href="https://beispiel.de"`).

#### 2. **Tags und Attribute**
HTML besteht aus **Tags**, die Inhalte umschließen und ihre Funktion definieren. Wichtige Tags für den Einstieg:
- `<h1>` bis `<h6>`: Überschriften (h1 = größte Überschrift).
- `<p>`: Absatz.
- `<strong>` oder `<b>`: Fettgedruckter Text.
- `<em>` oder `<i>`: Kursiver Text.
- `<img>`: Bild einbinden (mit `src`-Attribut für die Bildquelle).

**Attribut** = Zusatzinformation zu einem Tag, z. B.:
- `src="bild.jpg"` (für Bilder).
- `alt="Beschreibung"` (Barrierefreiheit).

#### 3. **Struktur einer HTML-Datei**
Eine einfache HTML-Datei hat folgende Grundstruktur:
```html
<!DOCTYPE html>
<html>
  <head>
    <title>Titel der Webseite</title>
  </head>
  <body>
    <h1>Überschrift</h1>
    <p>Hier steht ein Absatz.</p>
  </body>
</html>

Unterrichtsidee:
- Schüler:innen erstellen eine "Visitenkarte" mit ihrem Namen, einem Bild und einem Link zu ihrer Lieblingswebseite.

Unterrichtsideen und Methoden: HTML praxisnah vermitteln

Einstieg: HTML spielerisch entdecken

Ziel: Neugier wecken und erste Berührungspunkte schaffen.

1. Hörverstehen mit dem Wortspeicher (Arbeitsblatt)

Das HTML Arbeitsblatt "HTML Lückentext mit Wortspeicher, Hörverstehen und Suchsel" eignet sich ideal für den Einstieg:
- Hörverstehen: Die Lehrkraft liest den Text vor, während die Schüler:innen die Lücken mit Hilfe des Wortspeichers füllen.
- Wortspeicher: Webseite, Seiten, Inhalte, Hypertexte, Besucher, Befehlen, Grundlagen.
- Fokus: Fachbegriffe wiederholen und verinnerlichen.
- Suchsel: Im Anschluss suchen die Schüler:innen die Begriffe im Buchstabengitter – eine niedrigschwellige Sicherungsmethode.

Differenzierung:
- DaZ/DaF: Der Wortspeicher unterstützt Schüler:innen mit Sprachförderbedarf.
- Leistungsstärkere: Sie können zusätzlich eigene Sätze mit den Fachbegriffen bilden.

2. Analogie: HTML als "Rezept für eine Webseite"

Vergleichen Sie HTML mit einem Kochrezept:
- Tags = Zutaten (z. B. <h1> = "Hauptüberschrift").
- Attribute = Mengenangaben (z. B. src="bild.jpg" = "1 Bild").
- Browser = Koch, der das Rezept umsetzt.

Aktivität:
- Schüler:innen erstellen ein "Rezept" für eine einfache Webseite (z. B. "Wie baue ich eine Seite über mein Haustier?").

Erarbeitung: HTML praktisch anwenden

Ziel: Schüler:innen erstellen eigene HTML-Snippets und sehen direkte Ergebnisse.

1. Mini-Webseiten mit dem Editor

Material:
- Einfacher Texteditor (z. B. Notepad++, VS Code) oder Online-Editor wie CodePen.
- HTML Arbeitsblatt als Vorlage für die Struktur.

Ablauf:
1. Grundgerüst vorgeben: Die Lehrkraft stellt eine leere HTML-Datei bereit (siehe Struktur oben).
2. Aufgabenstellung:
- Erstelle eine Webseite über dein Lieblingstier mit:
- Einer Überschrift (<h1>).
- Einem Absatz (<p>) mit 2–3 Sätzen.
- Einem Bild (<img> mit src und alt).
- Einem Link (<a>) zu einer Tier-Webseite.
3. Präsentation: Schüler:innen zeigen ihre Ergebnisse im Browser.

Tipp:
- Nutzen Sie Pair Programming: Zwei Schüler:innen arbeiten zusammen (eine:r schreibt den Code, die:der andere diktiert).
- Fehlerkultur: Typische Fehler (z. B. fehlende schließende Tags) gemeinsam besprechen.

2. HTML-Quiz oder Stationenlernen

Station 1: Tags zuordnen
- Karten mit HTML-Tags (<h1>, <p>, <a>) und Beispielen ("Überschrift", "Absatz", "Link") werden zugeordnet.

Station 2: Code korrigieren
- Ein fehlerhafter HTML-Code wird vorgegeben (z. B. fehlendes </p>). Die Schüler:innen korrigieren ihn.

Station 3: Eigene Webseite planen
- Schüler:innen skizzieren auf Papier, wie ihre Traum-Webseite aussehen soll (Überschriften, Bilder, Links).

Sicherung: Wissen festigen und transferieren

Ziel: Gelerntes wiederholen und auf neue Kontexte anwenden.

1. Suchsel und Kreuzworträtsel

Das HTML Arbeitsblatt enthält bereits ein Suchsel mit Fachbegriffen. Ergänzend können Sie ein Kreuzworträtsel erstellen (Tools wie PuzzleMaker).

Fragen für das Rätsel:
- "Wie heißt das Tag für eine Überschrift?" (<h1>).
- "Wofür steht das href-Attribut?" (Link-Ziel).
- "Welches Tag bindet ein Bild ein?" (<img>).

2. HTML-Tagebuch

Schüler:innen führen ein "HTML-Tagebuch" mit:
- Täglichen Einträgen: "Heute habe ich gelernt, dass <strong> Text fett macht."
- Screenshots ihrer Webseiten.
- Reflexionsfragen: "Was war einfach? Was war schwer?"

3. Projekt: Klassen-Webseite

Aufgabe: Die Klasse erstellt gemeinsam eine Webseite über ihr Schulprojekt (z. B. "Unser Schulgarten").
- Rollenverteilung:
- Redaktion: Texte schreiben.
- Design: HTML-Struktur erstellen.
- Bilderteam: Fotos machen und einbinden.
- Umsetzung:
- Jede:r Schüler:in erstellt eine Unterseite (z. B. "Tomaten pflanzen").
- Die Seiten werden mit Links verknüpft.

Differenzierung und Sprachförderung

1. Für leistungsschwächere Schüler:innen

  • Vorgegebene Code-Snippets: Schüler:innen füllen nur Lücken aus (z. B. <h1>______</h1>).
  • Visuelle Hilfen: Farbige Markierungen für Tags (z. B. <h1> in Rot, </h1> in Blau).
  • Partnerarbeit: Stärkere Schüler:innen unterstützen schwächere.

2. Für leistungsstärkere Schüler:innen

  • Erweiterte Tags: Einführung von <ul> (ungeordnete Liste), <ol> (geordnete Liste), <table> (Tabelle).
  • CSS-Basics: Einfache Formatierungen mit <style> (z. B. color: red; für rote Schrift).
  • Forschungsauftrag: "Wie funktioniert das <div>-Tag? Erstelle ein Beispiel."

3. DaZ/DaF-Förderung

  • Wortspeicher (wie im HTML Arbeitsblatt) nutzen und um Bildkarten ergänzen.
  • Satzbausteine vorgeben:
  • "Das Tag <p> macht einen ______." (Absatz)
  • "Mit <a href="..."> kann ich einen ______ erstellen." (Link)
  • Hörverstehen: Lehrkraft liest HTML-Code vor ("Öffnendes Tag h1, Text, schließendes Tag h1").

Häufige Fragen von Lehrkräften zu HTML im Unterricht

Wie kann ich HTML in Klasse 4–7 einführen, ohne dass es zu technisch wird?

HTML lässt sich spielerisch und ohne Programmierkenntnisse vermitteln, indem Sie:
1. Analogien nutzen:
- Vergleichen Sie HTML mit einem Bauplan (Tags = Bausteine) oder einem Rezept (Attribute = Zutatenmengen).
2. Visuell arbeiten:
- Zeigen Sie den Schüler:innen eine Webseite (z. B. klexikon.zum.de) und lassen Sie sie die Struktur beschreiben ("Wo ist die Überschrift? Wo der Link?").
3. Unplugged-Aktivitäten:
- HTML mit Klebezetteln: Schüler:innen schreiben Tags auf Zettel und kleben sie an die Tafel (z. B. <h1> + "Unser Schulausflug" + </h1>).
- Papier-Prototypen: Sie skizzieren eine Webseite auf Papier und beschriften die Elemente mit Tags.

Material-Tipp:
Das HTML-Arbeitsblatt mit Lückentext und Wortspeicher ist speziell für diese Altersgruppe konzipiert und führt schrittweise in die Fachbegriffe ein.

FAQ: Häufige Fragen von Lehrkräften

Warum sollte ich HTML im Deutschunterricht behandeln?

HTML eignet sich fächerübergreifend, weil es:
1. Medienkompetenz fördert: Schüler:innen verstehen, wie Webseiten aufgebaut sind – eine wichtige Fähigkeit in einer digitalen Welt.
2. Sprachliche Strukturen trainiert:
- Texte gliedern (Überschriften, Absätze) – analog zum Schreiben von Aufsätzen.
- Präzision: HTML erfordert klare Anweisungen (z. B. schließende Tags), ähnlich wie beim Formulieren von Arbeitsanweisungen.
3. Kreativität anregt: Schüler:innen gestalten eigene Inhalte (z. B. eine "Über mich"-Seite).

Beispiel aus dem Deutschunterricht:
- Thema "Bericht schreiben": Schüler:innen erstellen einen HTML-Bericht über ein Schulprojekt mit Überschriften, Absätzen und Bildern.

Brauchen Schüler:innen Vorkenntnisse in Informatik, um HTML zu lernen?

Nein! HTML ist keine Programmiersprache und erfordert keine Vorkenntnisse. Wichtig ist:
- Grundlegende Computerkenntnisse (z. B. Dateien speichern, Browser bedienen).
- Interesse an digitalen Medien – mehr nicht.

Tipp für den Einstieg:
- Beginnen Sie mit unplugged-Aktivitäten (z. B. HTML mit Klebezetteln an der Tafel).
- Nutzen Sie visuelle Editoren wie CodePen, die sofortige Ergebnisse zeigen.

Welche typischen Fehler machen Schüler:innen bei HTML – und wie kann ich sie vermeiden?

Häufige Fehler und Lösungen:

Fehler Ursache Lösung
Fehlende schließende Tags Vergessen von </p>, </h1> etc. - Farbliche Markierung (z. B. öffnende Tags in Grün, schließende in Rot).
- Checkliste ("Habe ich jedes Tag geschlossen?").
Falsche Schreibweise z. B. <H1> statt <h1> (Großbuchstaben) - Syntax-Highlighting im Editor nutzen (falsche Tags werden rot markiert).
Fehlende Anführungszeichen z. B. src=bild.jpg statt src="bild.jpg" - Beispiel vorgeben und gemeinsam korrigieren.
Bilder werden nicht angezeigt Falscher Pfad im src-Attribut. - Relative Pfade erklären (z. B. bilder/katze.jpg).
Links funktionieren nicht Falsches href-Attribut (z. B. href=google.de). - Vollständige URLs vorgeben (z. B. https://www.google.de).

Tipp:
- Nutzen Sie Fehler als Lerngelegenheit: Zeigen Sie einen fehlerhaften Code und lassen Sie die Schüler:innen ihn korrigieren.
```
